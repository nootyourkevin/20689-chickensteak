# VocaLand v2 重新设计

> 从"CET 词库驱动的翻译练习"转为"兴趣话题驱动的英文对话 + 触屏取词"
> 状态：设计文档，尚未开始编码

---

## 一、核心理念转变

| 维度 | v1（旧） | v2（新） |
|------|---------|---------|
| 词从哪里来 | CET-4/6 考纲词库 | 用户在对话中主动点击的生词 |
| 对话切入点 | 无明确入口，"聊日常" | 基于用户兴趣的新闻/话题摘要 |
| 对话语言 | 用户打中文 → AI 翻译 → 追踪英文词 | AI 全程英文，用户尽量英文，卡住可打中文 |
| 学习驱动力 | 系统自动扫描 CET 词并追踪 | 用户点击不认识的词 → 弹出释义 → 记录 |
| 复习方式 | 无独立复习入口 | 卡片式背词，SM-2 算法驱动 |
| 用户系统 | 单用户 | 多角色（每个角色有独立的水平+兴趣档案） |

## 四、数据模型变化

### 新增表

```sql
-- 角色/用户档案
CREATE TABLE user_profile (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,                    -- 角色名
    english_level TEXT NOT NULL,           -- beginner/primary/middle/high/advanced/fluent
    interests TEXT NOT NULL DEFAULT '[]',  -- JSON: ["ai_tech", "finance", ...]
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 角色会话记录（每次进入 Page 3 创建一个 session）
CREATE TABLE chat_session (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    topic_title TEXT NOT NULL,             -- 话题标题
    topic_source TEXT,                     -- 新闻来源/LLM生成
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    ended_at TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user_profile(id)
);

-- 角色生词本（跨会话累积）
CREATE TABLE user_vocabulary (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    word TEXT NOT NULL,
    session_id INTEGER,                    -- 首次遇到该词的会话
    lookup_count INTEGER DEFAULT 1,        -- 被点击查词次数
    state TEXT DEFAULT 'NEW',              -- NEW / LEARNING / MASTERED
    mastery_score INTEGER DEFAULT 0,
    last_reviewed_at TIMESTAMP,
    next_review_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user_profile(id),
    FOREIGN KEY (session_id) REFERENCES chat_session(id),
    UNIQUE(user_id, word)                  -- 同一个角色不重复记录同一个词
);
```

### 复用现有表

- `words` — 词典数据（6,664 词 + 释义 + 例句 + 短语），**只读查询，不写**
- `word_mastery` — 全局掌握度（可选保留）
- `learning_events` — 学习事件日志，改为关联 `user_id` + `session_id`

## 五、AI 提示词自适应

根据用户英语水平动态调整 Prompt 中的角色设定：

| 水平 | persona | 语速/用词 | 特殊行为 |
|------|---------|----------|---------|
| 零基础 (beginner) | 耐心的老师朋友 | 极简词汇，短句，<30 词/句 | 允许中英混合，主动解释生词 |
| 初级 (primary) | 鼓励型伙伴 | 基础词汇，中短句，<40 词 | 偶尔用中文确认理解 |
| 中级 (middle) | 平等对话朋友 | 日常+适度进阶词汇，<50 词 | 自然对话，仅在用户求助时解释 |
| 高级 (high) | 深度讨论伙伴 | 正常英语，<60 词 | 可讨论复杂话题 |
| 流利 (advanced) | 母语级对话者 | 无限制 | 自然交流 |

**实现方式**：`PromptBuilder` 接受 `english_level` 参数，从配置表中选取对应的 persona 设定。这部分**由代码规则决定**，不需要 LLM 决策（LLM 决策会增加不确定性）。

### 保留不变的模块

| 模块 | 文件 | 说明 |
|------|------|------|
| 词汇仓库 | `vocabulary_repository.py` | 查词 + 句子 + 短语，查词弹窗的核心数据源 |
| 状态机 | `state_machine.py` | 五阶段逻辑不变 |
| SM-2 | `sm2_srs.py` / `srs_scheduler.py` | 卡片复习的排期算法 |
| 掌握度 | `mastery_scorer.py` | 评分逻辑不变 |
| 学习事件 | `learning_event.py` / `learning_evaluator.py` | 保留但关联到 `user_id` |
| LLM | `llm/base.py` / `cloud_llm.py` / `mock_llm.py` | 不变 |
| TTS | `tts/` | 不变 |

### 需要重写的模块

| 文件 | 变化 |
|------|------|
| `main_window.py` | 从单窗口改为多页面导航 |
| `chat_bubble.py` | QLabel → QTextEdit（可点击取词） |
| `conversation_manager.py` | 去掉自动 CET 词扫描；词由用户点击驱动 |
| `prompt_builder.py` | "教 CET 词" → "基于话题自然对话 + 水平自适应" |
| `word_summary.py` | "目标词/用过/复习中" → "本次生词"（含释义+出现次数） |
| `target_word_tracker.py` | 不再从 LLM 回复"猜词"，改为追踪用户点击的词 |

### 新增的模块

| 文件 | 功能 |
|------|------|
| `ui/home_page.py` | Page 1：角色选择 + 初始化 |
| `ui/topic_feed_page.py` | Page 2：话题/新闻列表 |
| `ui/chat_page.py` | Page 3：两栏聊天界面 |
| `ui/review_page.py` | Page 4：卡片式生词复习 |

## 八、未决定/待讨论

1. **首页要不要密码？** — 你说的是"角色"不是"账户"，更像是切换身份。如果只是区分学习档案（不是安全目的），可以不加密码。

2. **多个角色之间生词是否共享？** — 建议不共享。每个角色有独立的生词本和复习队列。

3. **Page 2 的话题数量** — 每次生成几个？建议 5-8 个。

4. **MQTT/网络同步** — 后续是否需要多设备同步学习进度？
